Users Table

Rust structs:

pub struct User {
    pub id: i64,
    pub username: String,
    pub password_hash: String,
    pub role: UserRole,
    pub created_at: DateTime<Utc>,
    pub updated_at: DateTime<Utc>,
    pub is_active: bool,
    pub email: Option<String>,
    pub last_login_at: Option<DateTime<Utc>>,
}

pub struct NewUser {
    pub username: String,
    pub password_hash: String,
    pub role: UserRole,
    pub email: Option<String>,
}


SQLite table:

CREATE TABLE IF NOT EXISTS users (
    id              INTEGER PRIMARY KEY,
    username        TEXT    NOT NULL UNIQUE,
    password_hash   TEXT    NOT NULL,
    role            TEXT    NOT NULL,
    is_active       INTEGER NOT NULL DEFAULT 1,
    email           TEXT    UNIQUE,
    created_at      TEXT    NOT NULL,
    updated_at      TEXT    NOT NULL,
    last_login_at   TEXT
);


Notes:

id is an auto-incrementing primary key (maps to i64).

username is unique and required.

email is unique but may be NULL. SQLite allows multiple NULLs in a UNIQUE column, which is acceptable here.

role stores the string representation of UserRole (e.g. "admin").

is_active uses 0/1 internally.

created_at and updated_at are stored as ISO 8601 strings.

last_login_at may be NULL.

    pub last_started_at: Option<DateTime<Utc>>,
    pub j_max_memory_mb: u32,
    pub j_min_memory_mb: u32,
}

pub struct NewServerInstance {
    pub name: String,
    pub mc_version: McVersion,
    pub port: u16,
    pub rcon_enabled: bool,
    pub rcon_port: Option<u16>,
    pub j_max_memory_mb: u32,
    pub j_min_memory_mb: u32,
}


SQLite table:

CREATE TABLE IF NOT EXISTS servers (
    id               INTEGER PRIMARY KEY,
    name             TEXT    NOT NULL,
    slug             TEXT    NOT NULL UNIQUE,
    mc_version       TEXT    NOT NULL,
    port             INTEGER NOT NULL,
    rcon_enabled     INTEGER NOT NULL,
    rcon_port        INTEGER,
    j_max_memory_mb  INTEGER NOT NULL,
    j_min_memory_mb  INTEGER NOT NULL,
    created_at       TEXT    NOT NULL,
    updated_at       TEXT    NOT NULL,
    last_started_at  TEXT
);


Notes:

slug is a unique, filename/URL-safe identifier derived from name (e.g. lowercased with spaces replaced by '-').

mc_version stores the string representation of McVersion (e.g. "1.21.1").

port is the TCP port for the Minecraft server. There is no database-level uniqueness constraint on port.
Port conflicts will be checked at runtime only against currently-running servers.

rcon_enabled uses 0/1.

rcon_port is optional (NULL when no RCON is configured).

j_max_memory_mb and j_min_memory_mb are stored as integers representing megabytes.

created_at and updated_at are stored as ISO 8601 strings.

last_started_at may be NULL.
